## 修改需求

### 需求：全站UI主题系统与交互动效

系统必须支持白天/黑夜（浅色/深色）双主题，并能在运行时平滑切换且具备用户偏好记忆。系统界面需要深度结合现代简约设计与高级炫酷特效（例如背景全局光晕、卡片边框流光、按钮点击涟漪或缩放反馈），并提供精心打磨的弹窗（Toast）提示样式（含丝滑出现的贝塞尔动画与颜色状态反馈），确保整体用户体验沉浸且高效。**页面必须提供有效的 favicon 文件，避免 404 请求。所有模态框（确认弹窗、导入弹窗、来源定位弹窗等）打开与关闭时必须正确切换 `aria-hidden` 属性以保障无障碍可用性。前端主题切换、API 请求封装、Toast 提示、HTML 转义等公共逻辑必须抽取为独立的公共脚本模块，禁止在多个页面脚本间复制粘贴。**

#### 场景：记忆并应用主题

- **当** 用户此前切换至浅色模式后再次刷新页面或重新进入应用
- **那么** 系统读取并自动应用浅色模式相关的CSS变量和特效，保持用户的视觉偏好

#### 场景：炫酷交互反馈

- **当** 用户鼠标悬浮在记账列表卡片或提交按钮上，或发生错误提示弹窗时
- **那么** 元素呈现高级流光效果或流畅弹跳、变色动画，错误Toast明确且动画柔和不突兀

#### 场景：模态框无障碍属性同步

- **当** 用户打开或关闭导入弹窗、来源定位弹窗等模态框
- **那么** 对应元素的 `aria-hidden` 属性必须与可见性状态同步，屏幕阅读器不会读出隐藏的弹窗内容

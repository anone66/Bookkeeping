## 修改需求

### 需求：管理员用户列表与账号维护

系统必须提供**管理端**能力，使具备管理员角色的用户能够查看系统中**用户列表**（至少包含可识别的账号标识与启用状态），并能够**创建**新的普通用户账号（需设置登录名与初始口令或等价凭据）；必须支持对已有账号执行**禁用**、**启用**或**删除**操作，以使该用户无法再登录或从系统中永久移除（删除时须同时清除该用户的所有交易记录与会话数据）。管理员不得删除自身账号。普通用户必须不得执行上述管理操作。

#### 场景：管理员查看用户列表

- **当** 管理员已登录且具备管理员角色并打开管理端
- **那么** 系统展示当前所有用户的列表信息（至少含登录名与是否可登录的状态）

#### 场景：管理员创建用户

- **当** 管理员提交一个新用户的登录名与满足规则要求的凭据
- **那么** 系统创建该用户（默认普通用户角色），且新用户随后可通过用户端登录流程登录（除非被立即禁用）

#### 场景：管理员处置账号

- **当** 管理员对某一用户执行已实现的禁用或删除操作并确认成功
- **那么** 该用户无法再使用原凭据登录，或按删除语义从用户集合中移除且其数据策略符合设计约定

#### 场景：管理员删除用户及其数据

- **当** 管理员对某一非自身用户执行删除操作并确认
- **那么** 该用户记录、所有交易记录及会话记录从数据库中永久移除

#### 场景：管理员不能删除自身

- **当** 管理员尝试删除自身账号
- **那么** 系统拒绝该操作并返回明确错误

#### 场景：普通用户无法管理他人账号

- **当** 普通用户（非管理员）尝试访问管理端写接口或页面
- **那么** 系统拒绝该操作并返回未授权或等效结果

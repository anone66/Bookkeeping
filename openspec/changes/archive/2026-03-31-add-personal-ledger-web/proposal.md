# 变更：个人收支记账 Web（SQLite）

## 为什么

需要在本地浏览器中快速记录消费与收入、查看累计结果，并能为每笔账目补充说明；数据须可靠落盘，便于长期自用而不依赖外部服务。

## 变更内容

- 新增基于浏览器的记账界面：支持录入**消费**与**收入**金额，并展示累计消费、累计收入及净值（收入减消费）。
- 支持对**每一笔**交易填写或后续编辑**补充信息**（例如备注、分类说明等，具体字段以 `design.md` 为准）。
- 使用 **SQLite** 作为唯一持久化存储；应用启动时初始化库表，读写均通过服务端（或约定嵌入层）访问数据库，禁止仅靠浏览器临时存储承担主数据。
- 界面目标为**简洁、现代、有一定视觉层次**（渐变/卡片/轻动效等），交互路径短，不引入复杂业务模块。

## 影响

- **受影响规范**：新增能力 `personal-ledger`（见 `specs/personal-ledger/spec.md`）。
- **受影响代码**：当前仓库无实现；批准后将在本仓库新增后端（或全栈最小应用）、SQLite 文件路径配置、静态前端资源及启动说明。
- **非目标**：多用户与权限、云同步、复杂报表、移动端原生应用（可在后续独立提案中扩展）。

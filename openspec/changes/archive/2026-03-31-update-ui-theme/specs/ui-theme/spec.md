## 新增需求
### 需求：全站UI主题系统与交互动效

系统必须支持白天/黑夜（浅色/深色）双主题，并能在运行时平滑切换且具备用户偏好记忆。系统界面需要深度结合现代简约设计与高级炫酷特效（例如背景全局光晕、卡片边框流光、按钮点击涟漪或缩放反馈），并提供精心打磨的弹窗（Toast）提示样式（含丝滑出现的贝塞尔动画与颜色状态反馈），确保整体用户体验沉浸且高效。

#### 场景：记忆并应用主题

- **当** 用户此前切换至浅色模式后再次刷新页面或重新进入应用
- **那么** 系统读取并自动应用浅色模式相关的CSS变量和特效，保持用户的视觉偏好

#### 场景：炫酷交互反馈

- **当** 用户鼠标悬浮在记账列表卡片或提交按钮上，或发生错误提示弹窗时
- **那么** 元素呈现高级流光效果或流畅弹跳、变色动画，错误Toast明确且动画柔和不突兀

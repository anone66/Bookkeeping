## 需求
### 需求：用户登录与登出

系统必须允许用户通过**登录名与口令**建立已认证会话，并在用户主动登出或会话失效后结束访问受保护资源的能力。口令不得以明文形式持久存储于数据库中。登录失败时不得区分「用户不存在」与「口令错误」之外的敏感细节（可实现为统一错误提示）。

#### 场景：登录成功获得会话

- **当** 用户提供正确的登录名与口令且账号处于可登录状态
- **那么** 系统建立会话，随后的受保护请求在该会话有效期内被识别为该用户

#### 场景：登录失败

- **当** 用户提供错误或不可用的凭据
- **那么** 系统不建立会话，并返回与成功路径明确区分的错误结果

#### 场景：登出

- **当** 已登录用户执行登出操作
- **那么** 后续请求不再视为已认证（除非再次登录）

### 需求：受保护的收支访问控制

系统必须禁止**未认证**客户端访问创建、查询、修改或删除**本人以外或匿名**的收支数据的所有接口；此类请求必须返回未认证错误且不得在响应体中泄露其他用户的存在性或流水内容。

#### 场景：未登录不可记账

- **当** 客户端在未携带有效会话的情况下调用收支写入接口
- **那么** 请求被拒绝且不写入数据库

#### 场景：未登录不可读汇总与列表

- **当** 客户端在未携带有效会话的情况下调用收支汇总或列表接口
- **那么** 请求被拒绝

#### 场景：未登录不可导入账单

- **当** 客户端在未携带有效会话的情况下调用账单批量导入接口
- **那么** 请求被拒绝且不写入数据库

# 变更：体验优化与功能增强

## 为什么

随着用户数据量增长（尤其在导入数千条账单后），当前无分页的全量加载模式将导致接口响应缓慢与前端渲染卡顿。SQLite 未启用 WAL 模式，多用户并发时读写互斥导致锁等待。此外，用户反馈了多项功能缺失：无法导出数据、无法修改交易类型和日期、无法搜索交易、无法自助修改密码、管理员无法删除用户。前端在所有异步操作期间缺少加载指示器，Toast 无法手动关闭。这些问题影响了产品在实际使用场景中的可用性与用户满意度。

## 变更内容

### 性能优化
- 交易列表 API 增加分页参数（`page` / `page_size`），默认返回一页而非全量
- 前端实现滚动加载或分页 UI
- SQLite 连接启用 WAL 模式以提升并发读性能

### UX 改进
- 所有异步操作期间展示加载指示器（按钮禁用 + spinner / 顶部进度条）
- Toast 支持点击关闭
- 交易 PATCH 接口支持修改 `type`（消费/收入）和 `transacted_on`（交易日期）

### 新功能
- 新增数据导出接口与前端入口（导出当前筛选范围的交易为 CSV）
- 新增交易关键词搜索功能（按备注/对方/商品说明模糊匹配）
- 普通用户自助修改密码
- 管理员可删除用户

## 影响

- 受影响规范：`personal-ledger`、`ui-theme`、`admin-users`
- 受影响代码：`app/main.py`、`static/app.js`、`static/admin.js`、`static/index.html`、`static/admin.html`、`static/style.css`
- **重大变更**：交易列表 API 返回结构从纯数组变为包含分页元数据的对象

## 新增需求

### 需求：支付宝与微信账单导入

系统必须为**已登录的当前用户**提供从**支付宝交易明细 CSV**与**微信支付账单流水 XLSX**批量导入交易的能力；导入的每一行经校验后必须写入 SQLite，且**仅归属于该用户**，汇总与列表行为与手工录入一致。除金额、收支类型与交易日期外，系统必须将导出中与对账相关的**结构化字段**落入数据库**独立可空列**（手工录入的记录这些列为空）：支付宝侧包括交易分类、交易对方、商品说明、收/付款方式、商家订单号、导出文件中的备注列等；微信侧包括交易类型、交易对方、商品、支付方式、商户单号、导出备注列等；与同表已有的用户可编辑**补充信息**字段（`note`）区分——`bill_export_note` 仅承载导出备注原文或可解析片段，用户仍可在 `note` 中自行补充说明。列表或详情界面必须允许用户查看这些导入字段，以便与原始账单核对。导入必须通过**记账页**上的统一入口触发：用户点击导入后**下拉选择平台**（支付宝或微信），再选择本地导出文件并提交。服务端必须校验所选平台与实际上传文件格式一致（例如扩展名与表头特征），不一致时必须拒绝请求并返回明确错误。解析必须兼容本项目 `账单/` 目录下提供的官方导出样例结构：支付宝 CSV 在说明性前置行之后出现以「交易时间」为首列的明细表头；微信 XLSX 在说明性前置行之后出现含「交易时间」「收/支」「金额(元)」等的表头行，且「交易时间」列可能以 **Excel 日期序列**形式存储。业务映射规则：**「支出」→ 消费、「收入」→ 收入**；金额必须为正并成功解析为系统使用的货币最小单位；**必须跳过**支付宝「不计收支」与微信「中性交易」对应行；仅导入状态表明**交易已成功完成**的明细（例如样例中的「交易成功」「支付成功」「已转账」等，以实现与固定样例对齐为准）。**幂等**：对同一用户、同一导入来源标识、同一外部交易单号（支付宝「交易订单号」、微信「交易单号」）不得重复插入第二条记录；重复提交同一文件或交叉重复时必须稳定地计入已存在或跳过，而不 inflate 汇总。

#### 场景：支付宝 CSV 导入成功

- **当** 已登录用户选择支付宝平台并上传与样例一致的有效明细 CSV
- **那么** 应写入若干条消费/收入记录（不计收支行不落库），并可在该用户列表与汇总中看到相应更新，且不包含其他用户数据

#### 场景：微信 XLSX 导入成功

- **当** 已登录用户选择微信平台并上传与样例一致的有效流水 XLSX
- **那么** 应写入若干条消费/收入记录（中性交易不落库），交易日期与金额与导出内容一致（时区按导出说明处理），并正确反映在该用户列表与汇总中

#### 场景：平台与文件不匹配

- **当** 用户选择支付宝却上传 XLSX，或选择微信却上传 CSV，或文件表头与所选平台不符
- **那么** 服务端必须拒绝导入且不写入半成批数据，并返回可理解的错误说明

#### 场景：重复外部单号不重复入账

- **当** 同一用户再次导入包含已入库外部单号的文件
- **那么** 不得为相同单号新增重复交易，汇总金额不得被重复计算

#### 场景：刷新后导入数据仍在

- **当** 导入成功后用户刷新页面或重启服务后重新登录
- **那么** 已导入交易仍可从 SQLite 加载并仅对该用户可见

#### 场景：导入字段可查阅与补充信息并存

- **当** 用户已导入一笔交易并在记账列表或详情中查看该笔
- **那么** 应能区分展示账单来源字段（分类/类型、对方、商品摘要、支付方式、商户单号、导出备注等）与用户 `note` 中的补充说明；手工录入的交易不展示空的账单列或应折叠为空状态

## 修改需求

### 需求：简洁现代的浏览器界面

系统必须提供**简洁、现代**的页面布局与视觉层次（例如清晰的排版、统一的配色与卡片式分区），交互路径短。界面必须支持深浅色（白天/黑夜）模式切换，并在视觉上结合高级光效等炫酷特效。普通用户的记账视图须在**已成功登录**后提供；允许单独提供登录表单页或在首屏内嵌登录，但不得在未满足认证要求的同一视图内将匿名用户与已登录用户的记账数据混用。管理端界面可与用户端区分入口，但必须保持一致的可用性与清晰度。

#### 场景：登录后完成录入与查看汇总

- **当** 普通用户完成登录并进入记账视图
- **那么** 在同一视图或一步可见范围内能够完成录入表单并看到**本人**当前汇总，无需多步无关向导

#### 场景：管理端可达且受控

- **当** 管理员访问管理端入口
- **那么** 管理员可完成用户维护相关操作；非管理员访问该入口时不得获得写权限

#### 场景：主题切换与视觉特效

- **当** 用户点击主题切换按钮或根据系统偏好
- **那么** 界面可以平滑切换深色与浅色主题，并在卡片、按钮与背景上展示对应的高级流光与毛玻璃特效

#### 场景：从记账页导入第三方账单

- **当** 已登录用户在记账页使用导入入口、选择平台并提交导出文件
- **那么** 用户可在同一记账视境内触发导入并看到结果反馈（成功摘要或错误），且导入完成后列表与汇总反映新数据

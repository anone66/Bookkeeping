## 新增需求
### 需求：移动端自适应布局与交互优化

系统必须适配移动端视口并优化交互体验。在窄屏幕下（如移动设备），汇总卡片需调整为合理的空间排布；顶部导航需精简以节省空间；“新增一笔”等表单需从右侧边栏转换为悬浮按钮（FAB）加底部抽屉（Bottom Sheet）的形式；日期筛选与各项操作弹窗需改为从底部滑出。布局留白与列表排版应更紧凑，确保数据可见性与点按操作的便利性。

#### 场景：移动端隐藏表单与使用底部抽屉
- **当** 用户在屏幕宽度较小的移动设备上查看记账页时
- **那么** 默认右侧的新增记录表单应隐藏，取而代之的是右下角的悬浮按钮（FAB）；点击该按钮时，新增表单从底部以抽屉（Bottom Sheet）的形式滑出。

#### 场景：移动端空间利用与卡片重排
- **当** 用户在移动设备上查看财务概览时
- **那么** Bento 汇总卡片应重新排布（例如突出净值跨列展示，消费与收入并列），列表间距变小，顶部导航在视觉上精简，所有弹窗（如日期筛选、账单导入、来源定位等）均从屏幕底部滑出以便于操作。

## 修改需求

### 需求：全站UI主题系统与交互动效

系统必须支持白天/黑夜（浅色/深色）双主题，并能在运行时平滑切换且具备用户偏好记忆。系统界面需要深度结合现代简约设计与高级炫酷特效（例如背景全局光晕、卡片边框流光、按钮点击涟漪或缩放反馈），并提供精心打磨的弹窗（Toast）提示样式（含丝滑出现的贝塞尔动画与颜色状态反馈），确保整体用户体验沉浸且高效。**Toast 弹窗必须支持用户手动点击关闭，关闭后自动消失定时器须被取消。** **所有异步操作期间（登录、记账、导入、刷新列表等）必须展示加载指示器（如按钮禁用状态 + spinner 或顶部进度条），防止用户重复操作。**

#### 场景：记忆并应用主题

- **当** 用户此前切换至浅色模式后再次刷新页面或重新进入应用
- **那么** 系统读取并自动应用浅色模式相关的CSS变量和特效，保持用户的视觉偏好

#### 场景：炫酷交互反馈

- **当** 用户鼠标悬浮在记账列表卡片或提交按钮上，或发生错误提示弹窗时
- **那么** 元素呈现高级流光效果或流畅弹跳、变色动画，错误Toast明确且动画柔和不突兀

#### 场景：Toast 可手动关闭

- **当** Toast 弹窗显示时用户点击该弹窗或其关闭按钮
- **那么** Toast 立即隐藏，自动消失定时器被取消

#### 场景：异步操作展示加载状态

- **当** 用户触发登录、记账、导入或刷新等异步操作
- **那么** 对应的提交按钮处于禁用状态并显示加载指示器，直到操作完成或失败

## 修改需求

### 需求：移动端自适应布局与交互优化

系统必须适配移动端视口并优化交互体验。在窄屏幕下（如移动设备），汇总卡片需调整为合理的空间排布；顶部导航需精简以节省空间；"新增一笔"等表单需从右侧边栏转换为悬浮按钮（FAB）加底部抽屉（Bottom Sheet）的形式；日期筛选与各项操作弹窗需改为从底部滑出。布局留白与列表排版应更紧凑，确保数据可见性与点按操作的便利性。**交易列表在分页模式下须适配移动端的滚动加载或「加载更多」按钮交互。**

#### 场景：移动端隐藏表单与使用底部抽屉

- **当** 用户在屏幕宽度较小的移动设备上查看记账页时
- **那么** 默认右侧的新增记录表单应隐藏，取而代之的是右下角的悬浮按钮（FAB）；点击该按钮时，新增表单从底部以抽屉（Bottom Sheet）的形式滑出。

#### 场景：移动端空间利用与卡片重排

- **当** 用户在移动设备上查看财务概览时
- **那么** Bento 汇总卡片应重新排布（例如突出净值跨列展示，消费与收入并列），列表间距变小，顶部导航在视觉上精简，所有弹窗（如日期筛选、账单导入、来源定位等）均从屏幕底部滑出以便于操作。

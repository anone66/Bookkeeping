## 新增需求

### 需求：交易发生时间可选且默认当天

系统必须允许已登录用户在录入交易时指定“交易发生时间”；当用户未主动选择时，系统必须自动使用当天日期作为默认值并参与后续查询与统计。

#### 场景：未选择交易时间时自动默认当天

- **当** 当前登录用户提交一笔交易且未修改交易时间字段
- **那么** 该交易的交易发生时间应保存为提交当日日期，并可在记录详情中查看

#### 场景：手动选择历史日期后按所选日期入账

- **当** 当前登录用户在录入时手动选择某一历史日期并提交交易
- **那么** 该交易必须按该历史日期参与年份月份筛选、分组与汇总计算

### 需求：交易记录按年份月份分组与总体对照汇总

系统必须支持按年份、月份对当前登录用户的交易记录进行筛选和分组展示，并在展示分期汇总的同时保留全量总体汇总以便对照；任一展示或统计结果均不得包含其他用户交易。

#### 场景：按年份查看分组与汇总

- **当** 当前登录用户选择某一年份查看交易记录
- **那么** 系统应展示该年范围内按月份分组的交易列表，并返回该年消费、收入与净值汇总

#### 场景：按年月查看当月分组与汇总

- **当** 当前登录用户选择某一年份与月份查看交易记录
- **那么** 系统应仅展示该月交易，并返回该月消费、收入与净值汇总

#### 场景：分期汇总与总体汇总并行展示

- **当** 当前登录用户已选择任意年份或月份筛选条件
- **那么** 页面中应同时可见筛选范围汇总与总体汇总，且两者统计口径分别对应筛选范围与全量范围

# 移动端自适应设计

由于记账系统当前设计为分栏（split）结构与卡片式概览（Bento），在手机屏幕下呈现单列时存在如下挑战：
1. 记账表单被挤到最底部，记录时必须滚动穿过整个账单列表。
2. 数据汇总卡片单列叠加占据首屏大量空间。

为解决上述问题，本方案采用以下设计模式：
- **悬浮按钮（FAB）**：遵循典型的移动端添加操作模式，固定于页面右下角。
- **底部抽屉（Bottom Sheet）**：相比居中弹窗，更容易单手触达和关闭，在移动端用于展示“新增记录”及各种“筛选/信息面板”。
- **CSS Grid 重排**：使用媒体查询将 Bento 组件转为 2 列结构，突出净值（占满 2 列）。

技术实现上完全通过 CSS 媒体查询（`@media (max-width: 768px)` 等）及少量 JavaScript 切换 class 来实现，不依赖外部库。

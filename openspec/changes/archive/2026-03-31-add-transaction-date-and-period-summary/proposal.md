# 变更：新增交易时间选择与按年月分组统计

## 为什么

当前交易记录以累计汇总与时间倒序列表为主，难以满足按自然时间维度复盘和对账的需求。用户需要在录入时明确指定交易发生时间，并在查看时按年份、月份快速分组与核算，同时保留全量总体汇总用于全局对照。

## 变更内容

- 新增交易录入时间字段：用户在新增交易时可选择交易发生时间；未主动选择时默认当天日期。
- 新增按年份、月份查看能力：交易记录支持按年、按月筛选与分组展示，便于分期查看流水。
- 新增分期汇总能力：在所选年份或月份范围内，展示该范围消费、收入与净值；同时保留全量总体汇总用于对照。
- 明确时间语义：统计分组与筛选基于“交易发生时间”，而非系统创建时间。

## 影响

- **受影响规范**：`specs/personal-ledger/spec.md`
- **受影响代码**：后端交易数据模型与查询接口、前端录入表单与流水列表展示、汇总统计逻辑
- **兼容性说明**：历史数据若缺少交易发生时间，需要提供迁移或默认规则（例如回填为创建时间的日期部分）

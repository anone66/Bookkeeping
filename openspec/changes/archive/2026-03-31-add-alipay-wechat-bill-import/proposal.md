# 变更：支付宝与微信账单导入

## 为什么

用户已在支付宝、微信支付中沉淀大量历史流水，手工逐笔录入成本高。通过在记账页支持官方导出文件（CSV / XLSX）批量导入，并与现有 SQLite 多用户模型、交易日期与汇总逻辑对齐，可显著提升记账效率。

## 变更内容

- 在普通用户记账界面增加**导入**入口：点击后下拉选择**支付宝**或**微信**，再选择本地导出文件并提交；仅**已登录**用户可为**本人**导入。
- 服务端解析：支付宝按「交易明细」CSV 格式（含前置说明行与表头 `交易时间,…,收/支,金额,…`）；微信按「微信支付账单流水」XLSX 格式（前置说明行 + 明细表头含 `交易时间`、`收/支`、`金额(元)` 等；交易时间列可能为 Excel 序列日期）。
- 业务规则：将「支出 / 收入」映射为系统消费 / 收入；**跳过**支付宝「不计收支」与微信「中性交易」对应行；仅导入状态为已成功完成的明细（具体状态词以实现与样例对齐）；对同一用户、同一平台、同一外部单号**去重**，避免重复导入产生重复账本记录。
- **数据库**：为支持幂等导入，增加 `import_platform`、`external_id` 等字段；并将导出中的**结构化明细**（分类/类型、对方、商品摘要、支付方式、商户单号、导出备注等）落入**独立可空列**，与原有 `note`（用户补充说明）区分或在导入策略中明确合并规则（见 `design.md`）。手工录入记录上述列均为空，行为不变。

## 影响

- 受影响规范：`personal-ledger`、`user-auth`
- 受影响代码（实施阶段）：`app/main.py`（迁移、导入 API）、记账前端模板或静态资源、依赖（XLSX 解析库）、`tests/` 导入与权限测试
